HOISTING 
-----------
Hoisting is JavaScriptâ€™s default behavior in which all declarations move to the top during memory creation phase
eg. console.log(a+10)   -> NaN
    console.log(a)   -> undefined
    var a = 10
here, var a is declaration so it moves to the top but its initialization donot so it gives undefined nd nan 

JS runs the code twice line by line 
Global Execution Context -> 2 phases(memory creation, code execution)
memory creation                 code execution
var a = undefined               console.log(a+10) -> nan
                                console.log(a)  -> undefined 

Are let nd const variables hoisted?
- yes, let nd const variables are also hoisted but they are not initialized during memory creation phase. 
Its value is initialized during code execution phase.
eg.
clg(a);
let a = 10;
